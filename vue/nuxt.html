# 为什么要服务端渲染

  • 服务端渲染是前端发送请求，服务端响应后生成html响应。客户端渲染是响应一个json，浏览器拿到后再解析生成DOM树生成页面
  • SEO需求。如果都是向后端发送AJAX请求，这些请求是异步的，而爬虫将不会等待异步请求结束再抓取。如果需要SEO，就可以用服务端渲染
  • 首屏加载快。只需要响应一个渲染好的HTML，代码量少

# 有什么缺点

  • 生命周期钩子需要同时能够运行在服务端和客户端，所以某些第三方库需要特殊处理
  • 服务端渲染需要Node.js作为中间层。部署Vue项目时只需要依赖nginx环境。而服务端渲染的项目还需要Node.js的环境

# Nuxt.js的工作原理

  1. 浏览器发送HTTP请求到Node.js服务端
  2. Node.js服务端向Java Servlet服务接口发送数据请求
  3. Servlet响应JSON数据给Node.js服务端
  4. Node.js服务端进行渲染，生成html
  5. Node.js服务端将HTML响应给浏览器

# Nuxt.js的操作流程

  1. 收到请求，启动服务
  2. 运行中间件
  3. 校验参数
  4. 异步处理数据，向Java Servlet端发送数据请求
  5. 渲染

渲染完成后，就轮到浏览器页面的Vue的生命周期，beforeCreate和created，这两个部分也是在服务端执行

# 异步加载数据

利用aysncData（）进行接口访问，数据加载

  • 安装：npm i @nuxtjs/axios
  • 在config.nuxt.js中的modues属性引入
  • 在pages目录下的vue文件中创建方法：asyncData（{$axios}），这个方法会在加载组件之前，被服务端调用。方法内部：调用$axios.$get(接口地址)，会得到一个Promise对象，这个对象调用then（）在回调中获取response.data，回调中还要返回一个对象X，包括data属性。最后asyncData方法返回then的执行结果。因为then的回调中返回了一个非Promise对象的值，所以then的结果是一个Promise对象，状态是成功，值就是那个X。可以看到then还是太麻烦了，还是用async和await好，直接获取到response，然后返回X就好
  • 在template中就可以使用X中的属性

# 封装请求接口的api

  • 创建api目录
  • 暴露方法，参数为（{$axios}，inject），方法内部调用inject（要注入的方法名，data=>$axios.$get(接口地址)）。这一步把方法注入到context、vue、store。可以理解为inject的第一个参数是第二个参数的名称
  • 在plugins中添加这个插件文件
  • 这时候服务端已经有了这个方法，在asyncData（）参数中添加一个app，获取服务端。然后就可以调用方法
  • 客户端要调用就写mounted钩子

# 路由

  • 不需要单独配置路由，pages目录下的文件名就是路由地址

  • 路由跳转：用nuxt-link标签，to属性指定路径，要加斜杠

  • 动态路由：路径中有冒号的就是动态路由，对应文件中的目录下的_xxx.vue或者_xxx目录下的index.vue

  • 404的路由：用_.vue来编写404的页面

  • 中间件：是一个函数。相当于路由守卫，用于权限判断。放在middleware目录，文件名就是中间件名称。函数参数有context（包括一些方法、store等）。需要用到中间件就在pages下的vue文件的script中写middleware。如果希望用到全部页面，可以在nuxt.config.js，在router属性下写middleware属性

# 登录功能

因为登录网站不需要SEO，所以直接用普通的Vue写另外一个页面即可。从SSR的页面点击后跳到这个页面，登陆成功后返回原网页

登陆后会返回cookie信息，包括userInfo、token、refresh_token，则跳回原页面后需要用vuex保存登录信息。Nuxt.js的Store直接在store目录下创建js文件就是一个模块，Nuxt会自动管理

state需要是一个函数，避免返回引用类型，和Vue的date要用一个函数一样。然后编写mutations用于改变状态、actions定义行为

  • state：包括userInfo、token、refresh_token
  • mutation：一个函数reset负责设为null
  • actions：
    • 登录：跳转登录页操作，包括调用reset，和跳转到登陆页面。用window.location.href，并且被跳转的网站的参数后面还要加上redirectURL=window.location.href，在登录成功后返回点击登陆前的地址。这里用到了window，只有浏览器有window这个对象，所以只能在客户端调用这个action
    • 登出：reset、然后跳转回登陆页面进行logout，删除服务器登录数据后再返回原页面

# 页面编写

因为pages目录下的文件自动会配置好路由，所以直接在pages目录下创建页面即可。正常编写页面的样式结构，发送ajax请求的回调写在api目录下，用inject方法去注入，然后在actions中编写方法调用这些请求接口的方法
