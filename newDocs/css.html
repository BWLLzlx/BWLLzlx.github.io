# 盒子模型

盒子组成：content、padding、border、margin

  • 标准盒子：content-box，width 只算 content 的
  • 怪异盒子：border-box，width 算到 border，所以如果加大 padding 就会加入内边距

# 选择器

分类

  • id 选择器（#box），选择 id 为 box 的元素
  • 类选择器（.one），选择类名为 one 的所有元素
  • 标签选择器（div），选择标签为 div 的所有元素
  • 后代选择器（#box div），选择 id 为 box 元素内部所有的 div 元素
  • 子选择器（.one>one_1），选择父元素为.one 的所有.one_1 的元素
  • 相邻同胞选择器（.one+.two），选择紧接在.one 之后的所有.two 元素
  • 群组选择器（div,p），选择 div、p 的所有元素

优先级：

  • 根据来源：内联样式>内部样式>外部样式>浏览器用户自定义样式>浏览器默认样式

  • 根据分类：内联、ID、类、标签。分别计算四个的值，然后比较就行了，不进位

继承：

  • 字体
  • 文本
  • visibility

不可继承：

  • display
  • 文本属性：vertical-align、text-decoration
  • 盒子模型的属性：宽度、高度、内外边距、边框等
  • 背景属性：背景图片、颜色、位置等
  • 定位属性：浮动、清除浮动、定位 position 等
  • 轮廓样式属性：outline-style、outline-width、outline-color、outline

# 布局实现

# 水平居中

  • 行内：text-align：center
  • 块：
    • margin：0 auto，同时指定元素宽度
    • 父设置 flex，justify-content：center
    • 子绝父相，left 设置 50%，transform（translateX（-50%））

# 垂直居中

  • 文本：line-height 设置成 height
  • 块：定位+translateY、flex

# 浮动

外边距：垂直的话，下外边距不会影响自己的位置，不过会让自己的相当于底边位置改变，从而可以实现让其他盒子和自己对齐/居中的效果

外边距塌陷：正数和正数取大，负数和负数取绝对值大，正数和负数取相加的和

清除浮动的塌陷：

  • 在浮动结束的位置加上一个块元素，并且加上属性 clear:both，让浏览器知道哪个地方停止浮动
  • 父元素创建 BFC：BFC 可以包住浮动的元素
    • 例如通过父元素设置 overflow，可以是 auto
    • float
  • 伪元素：设置 content 为空，设置为 block，其他和第一种一样

# 定位

  • absolute：完全脱离，相对于他爹
  • relative：位置还在。相对于自己位置
  • fixed：固定在屏幕的位置，指定的 left 和 right 就是视口。会导致其他元素位置的变化
  • sticky：像 relative 和 fixed 的结合。会定在某个位置

# 两栏布局

左边固定,右边自适应。父，子 1，子 2

  • flex：父 flex，子 1 设置固定宽度，子 2flex1
  • 浮动：子 1 浮动，子 2margin-left 设置子 1 宽度
  • 定位：子 1 绝父相，子 2margin-left 设置子 1 宽度
  • 定位 2：子绝父相，left 一个 0 一个宽度

# 上中下，中又分左中右

  • flex：略
  • 浮动：并且是中间写在最上面
    • 三个东西浮动
    • 清除浮动
    • 中间的宽度设为 100%，左边就会被挤下去
    • 左边的东西 margin-left：设置了就会基于原位置移动。设置成-100%，就回到父的起点了。再加个相对定位就好
    • 右边的东西同样是加个 margin-left 和相对定位就好

# 媒体查询与响应式

媒体设备：

  • 分类：all、screen、print、speech。支持用逗号写多个
  • 指定适用于设备：style 指定 media 属性、link 指定 media 属性、@media 后面接分类接大括号，里面直接写样式

媒体查询的条件：

  • 与：and
  • 或：用逗号，不是 or
  • 非：not 加在@media 后面
  • only：能用媒体查询就当不存在，不能用就不用这个样式

理解：使用 @media 查询，可以针对不同的媒体类型、不同的尺寸设置不同的样式

# 移动端适配

逻辑像素与物理像素：

  • 在 css 写的 px 是逻辑像素。例如一个手机宽的逻辑像素只有 400，那么当一个 div 的宽度为 400px，会占满手机屏幕
  • 实际的像素点就是物理像素，好比那台手机宽的物理像素为 1200 个，那就是用 9（3x3）个物理像素呈现 1 个逻辑像素，相当于 div 的宽度变成了 1200px。物理像素越多，画面呈现效果就越细腻，就相当于 mc 的格子更多，所以通常会有高物理像素。但是会有一个问题如果一个图片的宽度原本是 50 个逻辑像素，在电脑就是用 50 个物理像素来显示的，但是到了手机1 个逻辑像素会被 3 个物理像素呈现，就会导致一个格子的内容被拉长成 3 个格子去显示，看起来就很糊
  • 怎么解决逻辑像素被多个物理像素呈现导致图片变糊：图片资源用更大的图片，例如假如要放 100px _ 100px 的图片，这原图就得是 300px _ 300px 的，这样电脑上看还是 100px _ 100px 的效果，手机看的是 300px _ 300px，但是因为原图就是 300px 的

px 与 em 与 rem：

  • px 是固定的像素
  • em 是相对于其父元素来设置字体大小
  • rem 是相对于根元素的 font-size，可以改 html 的 font-size 来改变 rem

vw 与 vh：这是动态的单位，会随着视口的宽度和高度进行变化，所以非常适合用来做移动端适配。我最喜欢的就是用 vw 和 vh 配合 max-width 和 min-width 进行响应式设计

# 字体样式

# line-height

表示两行字基线之间的距离。所以 line-height - font-size 也就是行距。开发中通常会给 body 设置一个 line-height，但是字体肯定不全部一样大，就会有些时候 line-height - font-size 为负，就嵌在一起了

值：

  • 百分比：表示
  • 数字 px：就是固定值
  • 数字：相对于当前元素的 font-size 的倍数。

# 溢出隐藏成省略号

单行：

  • overflow:hidden 隐藏溢出
  • text-overflow:ellipsis 溢出用省略号表示
  • white-space:no-wrap 强制显示在一行

多行：

  • 私有属性：
    • overflow:hidden 隐藏溢出
    • display: -webkit-box 类似于 flex
    • --webkit-box-orient: vertical 选择垂直排列
    • --webkit-line-clamp:数字 显示几行
  • 普通属性：使用伪元素，在前和后都加入伪元素
    • 前：content 用【...】，然后 absolute 固定到【右下角】
    • 后：content 用空，然后 absolute 到【右边】，注意不是右下角。然后 display 成行内块。因为是行内元素，所以会跟着最后一行的位置，在最右边。然后设置背景颜色和背景颜色一样，就能实现盖住省略号了。如果内容很多，这个盖子就会跟着内容被隐藏掉，省略号就显示出来了

补充：关于伪元素和伪类：伪元素只能有一个，伪类可以有好多个。伪元素只能在最后，伪元素在前面可以拼接多个

  • 伪类部分分类与部分类：
    • 状态类：link、hover、active、focus
    • 结构类：first-child、nth-child、nth-of-type
    • 表单类：checked、disabled

# 内容的显示与隐藏

display：none。none 就真没了，看不见摸不着

visibility：hidden。还存在。看不见摸不着但存在

opacity：0。只是设置了透明度。看不见摸得着

z-index：遮盖

position：直接移走
