# Let和Const

  • let用于声明变量
    • let不可重复声明同名变量，防止污染。var可以声明同名的变量
    • let声明的变量属于块级作用域，只能在代码块（if、for、while等）内有效。var没用块级作用域，不是函数内的话就是全局的变量。例子：for循环里面var i = 0，然后自增，给items[i]绑定单击事件，items[i].onclick = function(){this.xxxx}是没问题的，this是绑定这个事件的对象也就是items[i]。如果写成items[i].onclick = function(){items[i].xxxx}就有问题了，因为var声明的变量i是没有块级作用域的，所以这个i对象是在全局中存在的，遍历完之后i已经变成了n了。回调函数中需要i，根据作用域链先先找函数作用域，没有之后就去找全局作用域，但是找出来的i的值是n，所以根本没办法绑定，就算绑定也是不对的。如果换成let i = 0就行了，因为每个i都是在自己的块级作用域里面使用的，每次循环都是独立的块级作用域
    • 不存在变量提升，必须先声明后调用，规范
  • const应用于声明常量
    • 必须有初始值
    • 变量名潜规则用大写
    • 不允许重复声明
    • 块级作用域
    • 如果是引用类型的常量可以在不改变堆内存地址的情况下改变堆内存中存放的内容，因为栈内存存放的是堆内存的地址，修改堆内存的内容不会改变栈内存中常量的值

# 解构赋值

解构赋值是：按照模式匹配从数组、对象中提取值并且给变量赋值。属于变量声明语句

  • 数组 const ARRAY = [1,2,3,4]，解构赋值：let [a,b,c,d] = ARRAY，实现了给变量赋值
    • 如果模式不匹配（给的不是数组、或变量个数对应不上）就会失败且值是undefined。要2个给1个是解构失败，可以理解为没给的就默认给undefined，要1个给2个属于不完全解构是成功
    • 如果有初始值，只有赋的值是（===）undefined才能让初始值生效（只有赋值undefined或者不赋值的情况下，才会用初始值）。初始值可以调用解构赋值的其他变量，但是必须在它之前声明。初始值也可以调用函数，但是仅在赋值undefined或者不赋值的时候会去调用，不是的话直接用赋值了。
  • 对象 const LHC = { name:'lhc', fun :function(){alert（‘lhclhclhc’）} }，解构赋值：let {name,fun} = LHC。理解为创建一个假的对象并且name和fun去LHC对象里面复制值
    • 数组有下标有顺序所以变量名可以自由，但对象没有顺序所以变量名必须要和对象的属性或方法一致才能让属性找到变量
    • 对象的解构赋值的内部机制：变量中name的完整写法应该是name：name，前面的name是提供给对象匹配的模式，后面的name才是变量，用于赋值。对象找到同名的属性，然后给这个变量赋值。例如let {a：b} = {a:'123'}，对象找到和属性a同名的模式a的变量b，并且给b赋值‘123’。注意区分是模式还是变量
    • 目的是为了省事。如果常用Math对象的方法，想要向下取整要Math.floor（），通过解构赋值let {floor} = Math，就能获取到floor（）方法，直接floor（）就行
    • 例如是let {floor} = Math，最好不要先let x然后{x} = Math，因为**{开头容易被JS执行器理解为代码块**。可以在外面加上（）但是圆括号的使用要注意
    • 数组是特殊的对象，可以对数组解构赋值到一个对象的变量，格式为let {} = []。数组作为对象，也要根据匹配模式来找到变量，匹配模式是数组下标。例如 let {0:first, [arr.length-1] : last} = [ 1, 2, 3 ]，数组对象找到和0匹配的自己的属性1并把属性的值给变量first、3给last，实现了给变量赋值
    • 数组对象有属性length，可以像获取对象属性一样获取值赋给变量，let {length} = [1,2,3,4,5]，找数组对象[1,2,3,4,5]的和length匹配的模式的值赋给length
  • 字符串
    • 字符串被转化为类似于数组的对象。let [a,b,c] = "zlx"，实现了给变量赋值
  • 数值和布尔
    • 只要不是数组、对象，就转为对象，然后才给变量赋值
  • 函数
    • 传入参数1和2，就相当于let {x，y} = [1，2]
  • 圆括号什么时候能用：赋值语句的非模式部分，模式部分是指前面的那个

常用功能：

  • 利用数组的解构赋值进行数值交换，[a, b] = [b, a]
  • 获取json对象的数据
  • 提取函数返回值（函数返回数组/对象，用解构赋值来提取）
  • 在函数形参中使用，传递的实参是对象/数组，就能通过解构赋值获取某个属性，可以省事

# 箭头函数

  • 声明方式：let fun = () =>{}，相当于用 () =代替了funciton（）
  • this的指向是静态的，和声明该函数的作用域的this指向相同
  • 不能作为构造函数被调用，因为new实际操作是创建空Object对象，this指向这个对象然后调用构造函数给this的属性赋值，但是this不能指向那个object对象。
  • 不能用arguments伪数组
  • 简写：当参数只有一个可以不写（），当代码体只有一条语句可以不写{}，但是这样就不能写return，return的值就是语句的结果
  • 案例：事件绑定的函数中使用定时器，如果想修改绑定事件的对象的属性，没办法用定时器的内部匿名函数的this，因为这个函数是全局调用所以this指向window，但是如果定时器的函数用匿名函数，this指向声明该函数的作用域的this，那就是上层的this了
  • 适用场景：与当前this无关的回调

    • 【Symbol.内置属性】这么一个整体是对象的属性，自带的

    • 用于控制对象在特定场景下的表现

      例如Symbol.hasInstance。当某个类Person当被用来instanceof的时候（a instanceof Person），就会执行这个函数

# 迭代器Iterator

  • 迭代器是个接口也就是对象里的属性[Symbol.iterator](){}，实现这个接口用于实现遍历。ES6有for of循环，变量保存的是键值，for in保存的是键名

  • 原理：创建指针对象指向当前数据结构。指针对象有个next（）方法。第一次调用指向第一个成员，不断调用next（），指针移动直到最后一个。调用next（）返回一个对象，包括属性value和done

  • 自带迭代器的数据结构包括：map、set、数组等

  • 实际应用：想要对某个对象进行for of遍历，就要在这个对象内部写[Symbol.iterator](){}，返回一个指针对象也就是{}。指针对象内部要有next（）方法，并且返回一个对象，包括属性value和done，也就是在next（）方法return {value：xxx，done：false}。每次调用next方法，如果done是false就会继续调用，所以要在最后一次调用（index值等于当前对象的某个数组或者什么的长度）的时候return {value：undefined，done：true}。value的值就是当前下标的成员的值，并且每次返回对象之前要把下标index++

# 生成器函数Generator

  • 作用：异步编程

  • 声明：function * 函数名（）{}

  • 执行：不是函数名+（）的方式。生成器函数是迭代器对象，内部有next（）方法。所以执行是**用生成器函数对象调用next（）**方法

    • 内部有yield来分割函数体，1个yield分2段，2个分3段。yield '第一个分割线';
    • 每次调用next（）是执行一段函数体，next（）返回对象的value是当前代码段的下一个yield后面的表达式，会执行这个表达式计算结果
    • 因为生成器是迭代器对象，可以用for of循环，循环的变量就是相当于生成器函数调用next（）的value，就是yield语句中的表达式

  • 传参：调用next（）的时候可以传参。第二次调用next（）的时候传给第一个yield语句作为返回值

  • 实例：

    • 解决一直回调

      如果要1秒后输出111，然后2秒后输出222，然后3秒后输出333，常规是设置定时器，并且在内部再设置定时器，一直回调一直缩进

      有了生成器函数，分别写三个函数one（）、two（）、three（），内部是定时器的实现。然后再写一个生成器函数，内部写三个语句yield one（），yield two（）、yield three（）。在外部，生成器函数对象调用next（），就会执行一个代码段（为空），然后计算yield语句作为next（）方法返回的对象的value值，所以会执行one（）函数。调用完one（）之后，如果要让自动调用two（），不是在外部写生成器函数对象.next（），而是直接在one（）内部写：生成器函数对象调用next（）

    • 实现参数传递

      写三个函数，生成器函数内部yield语句+函数。在外部生成器函数对象调用next（），会执行生成器函数的第一个代码段（空），然后计算第一个yield语句的表达式结果（执行函数）。在函数1内写：生成器函数对象调用next（），并且传入参数，这个参数会作为yield语句的返回值，可以在代码段中接收。看着像是同步的代码（顺序执行），但是是异步执行

用法：数组去重、求交集（先去重，用filter遍历集合a的item，看集合b是否has（item），最后筛选出b中的a，就是b∩a）、并集（直接用...进行数组拼接，然后set去重）、差集（各自去重，用filter遍历a，看集合b是否has（item），最终筛选出b中的不是a的，就是b-a）

# Promise

  • Promise介绍与基本使用

    • Promise是什么

      Promise是个构造函数，用于异步编程（fs 文件操作、数据库操作mongodb、AJAX网络请求、定时器、事件），以前是靠单纯的回调函数

      通过Promise构造函数实例化对象时，需要传入参数：function（resolve，reject）{}。在这个执行器函数内部可以包裹异步操作。可以用到resolve（）和reject（）函数来给Promise实例化对象设置状态（对象有个属性promiseState，这两个方法就是修改这个属性的值）：初始化pending/成功resolved/失败rejected，调用resolve之后实例化对象的状态就是成功。还有属性promiseResult，在Promise对象的执行器函数中调用resolve（）和reject（）方法传入的参数可以设置为这个Promise对象的promiseResult属性的值（就是value或reason）。也就是说在.then（）中的value是调用这个方法的promise对象的promiseResult属性

      当实例化对象的状态为成功之后，可以用实例化对象调用then（）方法。then（）方法接收两个函数参数：前面一个函数表示成功，参数有value。后面一个函数表示失败，参数有reason。当状态为成功，执行第一个函数体；状态为失败，执行第二个函数体。then（）方法返回的是一个新的Promise对象

    • Promise的优点

      • 支持链式调用（非常重要），解决回调地狱不便于阅读和异常处理
      • 不需要提前写好回调函数。有了Promise是在启动了异步任务之后，返回了promise对象，才给它绑定处理。还可以绑定多个处理操作

    • Promise怎么用

      • 写一个方法，return的是Promise对象，在内部封装包裹异步操作实现功能。这样的话调用这个方法得到的就是一个Promise对象，可以直接调用then（）方法

      • util模块提供了promisify（）方法，用于将回调函数风格的方法自动转化为Promise风格的方法（返回Promise对象）

        const util = require（'util'）

        let newFun = util.promisify（某个函数）

        然后调用newFun（）就像调用那个函数一样，只是可以直接用.then（）

  • Promise API

    • Promise（）构造函数，构造函数的参数执行器函数是同步调用的，正常情况下都是先在执行器函数中改变状态，再指定回调

    • Promise.prototype.then（）方法，指定回调。方法是属于实例的。参数为一个回调函数

    • Promise.prototype.catch（）方法，指定失败情况下的回调。方法是属于实例的

    • Promise.resolve（）方法，返回一个状态为成功的Promise对象。这是属于构造函数的方法

      • 参数如果是非Promise对象的值，那就返回成功的promise对象
      • 参数如果是Promise对象，返回值取决于这个Promise对象的状态是成功还是失败

    • Promise.reject（）方法，无论参数是什么都返回状态为失败的Promise对象

    • Promise.all（）方法，参数是数组，数组元素是Promise对象

      • 当Promise对象都为成功状态，返回一个成功的Promise对象，这个对象的promiseResult是结果的数组。相当于给几个Promise对象做交集
      • 当有失败的，就返回失败的Promise对象，promiseResult是失败的数组

    • Promice.race（）方法，哪个Promise对象先改变状态为成功或失败，race（）返回的Promise对象就取决于谁

    • Promise.allSettled（）方法，参数是数组，数组元素是Promise对象，和all（）有点像，但是返回的必定是成功的Promise对象，值是Promise对象的数组

  • Promise关键问题

    • 如何改变Promise实例对象的状态

      • resolve（）

      • reject（）

      • throw xxx改为失败

    • 能否执行多个回调也就是调用几个then（）：只要状态发生了改变，就都会调用then（）方法

    • 先改变状态还是先指定回调函数then（）

      • 默认是先改后指定。指定的时候已经改完了，所以会调用then（）

      • 如果执行器函数封装的是异步操作，就会先指定后调。指定完还没改就不会调用then（），当状态改变就会调用then（）

    • then（）返回对象的状态

      由它回调函数的返回值决定，和resolve（）返回的结果相近。可以形成then（）链式调用，串联多个任务

      • 如果回调函数return非Promise对象的值，那就返回状态成功的promise对象。没写return语句返回undefined，则then（）返回的状态也是成功

      • 如果回调函数return的是Promise对象，取决于这个对象的状态是成功还是失败

      • 如果回调函数在then（）内部 throw xxx，返回失败的promise对象

    • 异常穿透

      • 使用then（）链式调用可以在最后指定失败的回调
      • 前面任何一步返回状态为失败的promise对象，都会传到失败回调处理

    • 中断Promise（）链：返回一个状态未改变的Promise对象，，则then（）返回的对象状态也是pending，则不会继续调用后续的then（）

  • Promise自定义封装

    • 构造函数，参数executor。在构造函数中调用，需要参数resolve和reject，这两个参数是函数，在构造函数内定义，并且这两个函数内部需要对PromiseResult和PromiseStatus进行修改，所以需要添加这两个属性，用this添加到Promise中，注意修改这两个参数的时候的this的指向，resolve（）和reject（）是由window进行调用的
    • 实例添加then（）方法，通过Promise.prototype设置
    • throw可以改变Promise对象状态为失败：因为是在函数executor中进行抛出的，所以要接收这个异常就是在构造函数中的调用executor的时候进行catch（）调用reject（）来修改对象状态
    • 只能改变一次状态：if（PromiseStatus ！== ’pending‘）就不能修改
    • then（）同步回调：通过判断状态是fulfiled还是rejected，执行onResolved（）或onRejected（）这两个参数的其中之一
    • then（）异步回调：当先指定回调函数后改变状态（例如改变状态的操作是计时器，那么先执行同步代码指定回调，再完成计时器操作改变状态，改变状态后再调用then（）的参数函数，但是作用域不一样，需要让resolve（）中能访问到then（）的参数），所以在then（）的方法内，在Promise对象的状态是pending的时候，把onResolved和onRejected这两个参数作为属性放到一个对象中保存起来，这个对象存到Promise构造函数中，然后在构造函数中的resolve()和reject（）中使用：if this.对象.onResolved（如果存在这个属性）就调用onResolved（）这个方法
    • 实现then（）的多个回调：如果是把onResolved（）和onRejected（）存到对象中，只能存一个对象，不能实现多个回调，谁在下面就实现谁。用数组存放对象，用push把对象加入数组，实现多个回调。调用时在resolve（）中用forEach（）遍历数组然后调用onResolved（）
    • 实现then（）的返回值：then（）的返回值是Promise对象，且和执行器函数的结果相关。return new Promise（），并且需要在执行器函数executor中获取回调函数的执行结果（then（）会传入两个回调函数一个解决成功一个解决失败，执行结果就是他们的返回值）进行分类，如果是Promise对象那么then返回的对象状态就和then内部回调函数的返回结果的状态有关，因为then内部函数返回结果是Promise对象，所以可以对这个Promise对象用then（）来分类，上面调用resolve（），下面调用reject（） 。不是Promise对象就设状态为成功也就是调用resolve（参数是回调函数的执行结果，因为then的return结果就是要和then的回调函数的结果相关）。并且记得要对throw之后的对象结果返回失败，也就是说要接住抛出的错误，哪里用了onResolved，哪里就接住并且catch里面reject（）
    • 异步状态下，原本是把onResolved这个参数直接作为属性加到Promise里面给resolve调用的（因为是resolve之后再执行then的语句），但是需要判断内部执行器函数的返回结果来决定then（）的返回结果，所以不能直接执行onResolved（）否则都是返回pending，需要封装一个函数来返回，获取内部函数的返回结果判断然后调用resolve（）设置then（）返回结果的状态，这样就不会是pending了
    • catch（）方法：直接用then（undefined，onRejected），但是需要在then（）return Promise对象之前，判断onRejected（），如果不是一个函数，那么给它初始化一个默认函数，也就是 onRejected = reason => {throw reason}，这样如果不传参数，也能相当于有个onRejected（）函数可以被使用。例如第一个then（）参数里面没有onRejected，保存起来的值就是undefined，当调用reject（）改变之后也会执行onRejected（）这么个语句，但是保存起来的值是undefined不是个函数，就会报错，后面的then也是走onRejected。如果有了默认的onRejected（）就不会影响后续的。onResolve（）也要这么弄一个默认的
    • Promise.resolve（）方法：返回一个Promise对象，判断resolve（）方法传递的参数，如果是Promise对象就用then（）调用resolve（）或reject（），如果不是就返回成功，也就是调用resolve（）
    • Promise.all（数组）：如果全部为成功就为返回一个状态为成功的Promiss对象，并且PromiseResult是所有值组成的数组，并且是按照顺序的。那么就遍历所有的Promise对象，调用.then（），上面就count++和arr[i] = value并且当count==length的时候就resolve（），下面就直接reject（）
    • Promise.race（数组）：遍历，直接then（）上面调用resolve（）和下面调用reject（）就好，因为resolve（）和reject（）已经设置了不能多次调用了，所以只要改了就只能是那一个
    • then（）方法是异步调用的，为了让它异步调用，需要在then（）方法的成功和失败的函数中加入定时器，不定时，但是实现异步
    • 如果要封装成类，写给实例的方法就是普通的：方法名（）{} ，如果是给类的方法也就是给Promise类的方法，就要加上static

  • async/await异步编程

    • async函数：返回Promise对象，Promise对象的状态和值与函数返回值有关。是Promsie对象就看Promise的状态和值，不是就是成功
    • await表达式：主要用于对await成功状态的值的获取
      • await要在async函数中
      • await右侧可以是Promise类型的数据。如果是Promise类型的数据，await操作会获取这个对象成功状态下的值。失败的会抛出错误，要try-catch
      • 如果右侧是其他类型的值，那就是什么给什么
    • 简洁，看着像同步但实际是异步。需要Promise类型的数据，就靠axios这个封装好的内容

# 模块化

好处：复用、维护、防止命名冲突

命令：

  • export暴露：
    • 分别暴露：直接写在要暴露的变量定义前面
    • 统一暴露：定义好全部变变量后，export{变量名1，变量名2……}
    • 默认暴露：export default 对象（这个对象里面封装要暴露的数据）

    • 解构赋值导入默认赋值的模块，因为它暴露的是对象，所以可以通过解构赋值接收：import {变量名1，变量名2} from "文件路径"，直接获得里面的变量。因为会有变量重名，可以用as指定别名
    • 简便方式：import 变量名 from "文件路径" ，只能用于默认暴露
