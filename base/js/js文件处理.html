# 文件类型

文件类型不同于文件后缀名，用于区别不同文件类型的是文件的编码方式。常见的文件类型有文本文件、二进制文件。文本文件的后缀名一般为txt，二进制文件的后缀名一般为bin，将文本文件的后缀改成另外的文本文件后缀，能够在一定情况下正常打开，但如果把文本文件后缀改为二进制文件的后缀就不行，是因为文本文件存储的是用ASCII码编码后的数据，如果改成了二进制文件的后缀名，解码的时候就是按照二进制文件的解码方式，而编码方式和解码方式不同，就会出现乱码

文本文件用【字符】作为单位来表示和存储数据，例如123这个值，文本文件将其视为字符‘1’和‘2’和‘3’，保存的是它们各自ASCII码的二进制表示。当记事本打开这个文本文件，读取二进制比特流，按照ASCII码来解码，8位二进制转成一个字符，最终组合在一起

二进制文件保存的是值的二进制值。例如123它保存的就是123的二进制值01111011

# 选择文件

使用input：file。通常会display：none，然后写一个另外的按钮A，并且在A的回调中，inputDOM对象调用click()，触发input：file的点击。multiple属性设置是否能上传多个文件。accept属性设置可以选择的文件类型，类型之间用英文的逗号隔开

# 上传文件

原理：使用ajax，通过FormData包装所有需要传递的参数。当使用axios发送post请求时，

API：

  • 构造函数：new FormData()，参数可以是一个表单，可以为空
  • 添加内容：FormData.prototype.append(name，value[，filename])，value可以是字符串，也可以是Blob。filename用于指定名称，如果是blob类型名称就是blob，如果是string类型名称是文件名称。可以给同一个name添加多个不同的值
  • 获取指定键的value：FormData.prototype.get()/getAll()，get()只能拿到第一个值，如果某个name有多个值，全拿要用getAll()
  • 删除指定键：FormData.prototype.delete()
  • 设置值【同名会覆盖】：FormData.prototype.set(key,value,[filename])
  • 判断有无指定键：FormData.prototype.has()
  • 获得迭代器对象：FormData.prototype.entries()，调用这个方法后可以获得一个迭代器对象，说人话就是一个数组。迭代器对象每次调用next()就会返回一条数据，输出对象包括done和value属性，键值对就是value属性【数组】。如果一个name有多个value，需要迭代多次。或者直接用for of去遍历迭代器对象
  • 获得迭代器的keys：FormData.prototype.keys()，也是相当于一个数组。与之相对的有FormData.prototype.values()

# 接收处理文件

模块需要：formidable

API：

  • new formidable.IncomingFrom()：创建form表单，解析formdata
  • formidable.prototype.uploadDir：设置临时文件存储位置
  • formidable.prototype.keepExtensions：设置临时文件的文件名是否包括扩展名
  • formidable.prototype.parse(req，（err，fields，files）=> { } )：解析请求中包含的form表单数据。fields存普通类型的数据，files存file类型的数据，files.file.filepath可以获取存储路径。配合fs模块的copy的API可以把文件存储在指定位置
