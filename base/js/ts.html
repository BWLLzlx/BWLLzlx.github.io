# 安装和使用

  • 下载ts：npm install -g typescript

  • 下载ts-node：npm install -g ts-node

  • 把ts编译成js并执行：tsc ts文件名，然后 node js文件名

  • 直接执行ts文件：ts-node ts文件名

  • 编译：tsc

  • 初始化生成配置文件：tsc -init

    • outDir指定编译后的目录
    • rootDir指定根目录

# parcel打包

  • 下载parcel：npm install parcel
  • 打包文件：parcel ./xxx/index.html

# 数据类型

  • 不指定数据类型就和js一样

  • 指定数据类型：在变量后加注解【:类型】，例如let a:boolean = false

  • ts的数据类型不可变，例如之前如果是number后面就不能是string

  • 联合数据类型：可以指定数据类型为【：number | string 】，用了这个通常还需要类型保护，例如在函数中参数是联合数据类型，就要区分数据类型，不同情况下做出不同的操作

  • 元组类型：【：[number, string] 】，规定每个元素的类型

  • 枚举类型enum：

    例如：enum Color {RED, GREEN, BLUE}，那么Color.RED是0，后续递增。Color[0]是‘RED’

    • 可以给RED指定例如RED = 3，那么后续从3递增
    • 可以给每个内容指定值，例如RED = “A”，要改就全改

  • 任何类型any：让变量和js一样，类型可变

  • 函数返回值也可以用【：数据类型】在函数名后指定，例如 function fun（）：string = {}，不写就是void

  • 类型断言：告诉ts我把这个变量看作什么类型

    • <数据类型>变量，告诉ts是把变量看作这个数据类型
    • 变量 as 数据类型，同样的效果

# 接口

# 创建自定义数据类型

功能：创建自定义数据类型，可以给变量指定属性

语句：

  • 普通数据类型：interface 接口名{ 属性 }
  • 可索引数据类型：interface 接口名 { [index：索引类型]：值类型 }

可选属性：在属性名后面加【?】可以在通过接口创建变量的时候不赋值这个属性

只读属性：在属性名前加readonly。属性用readonly，变量用const

使用：创建变量时正常使用

  • 普通数据类型：

    interface IUser { name：string，age?：number }

    const person : IUser = { name：‘zlx’ }

  • 可索引数据类型：

    interface IArr { [index : number] : string }

    const arr : IArr = [ "a"，"b" ]

    如果index的类型是string，就创建arr = { “a”："123"，“b”：“456” }

# 创建函数接口

功能：创建函数类型，可以给函数指定参数和返回类型

语句：interface 接口名 = { （参数：参数类型）：函数返回类型 }

使用：通过变量赋值声明函数

  • 创建函数的第一种方式：function xxx（）{}

    interface Itest = { (name：string)：void }

    function fun（）：Itest { }

  • 创建函数的第二种方式：let fun = （）=> {}

    interface Itest = { (name：string)：void }

    let fun : Itest = （name : string）：string => { return name }

    let name = fun（‘zlx’）

    consolo.log(name)

# 创建类接口

功能：创建类的类型，可以给类指定属性

语句：interface 接口名 = { 属性 }

使用：就和java一样创建类实现这个接口

  • interface IUser = { name：string，age?：number }

    class User implements IUSer { name：string，age?：number }

    let user1 = new User()

    user1.name = 'zlx'

# 接口继承

语句：interface 接口名 extends 父亲接口

# 混合接口

创建：接口内部可以同时写数据类型、函数类型

使用：可以把接口用给函数，指定函数的参数和返回类型；也可以把接口用给变量，指定变量的属性

# type

type定义的东西就理解成一种数据类型。接口只能定义为对象类型

type定义的东西不能重复定义。接口的可以重复定义，会视为合并在一起作为同一个接口

type定义的东西可以在其他type定义中用。interface不行

type定义的东西必须先声明后暴露。inteface支持同时

type的扩展用&操作符进行扩展。interface用extends

# 类

抽象类：

  • 创建：abstract class
  • 特点：不会被实例化。继承抽象类必须要实现抽象类的方法
  • 和接口的区别：抽象类中可以有普通的方法，接口中全是约束

类作为接口：类也可作为接口被接口继承

# 泛型

功能：方便用不同的类型的参数调用同一个函数，也可以限制参数全是同一个类型

备注：可以指定多个泛型，例如<T , S>

函数用泛型指定参数类型：

  • 定义：function fun<T> （参数名：T）: T { }，如果是数组可以用Array<T>，或 T[]
  • 使用：fun<string>（参数）

类用泛型指定参数类型：

  • 定义：class Person<T> { 属性：T }，也可以让T类型继承别的类型或者接口，也就是必须是这个类型
  • 使用：new Person<string>（参数）

# 命名空间

功能：划分作用域，避免命名冲突

语句：namespace xxx{ }

使用：在命名空间内用export暴露。引入时是【命名空间名 . 内容】，不需要import语句

# 在vue中使用typescript

创建项目：

  • 自定义创建，选上typescript支持

vue组件：

  • 在script部分，加上lang=ts
  • 主要使用type、interface、数据类型注解

vuex：

  • state：先创建接口State，规定有哪些属性以及类型。然后在引入state的时候指定state为State。state需要是State类型，但是不能用state:State的语句来指定类型，会被解析为值。state的类型还可以是（）=>State，所以写state : （）=> State，而()也要指定为State类型。是state: (): State => ({State每个属性的值})
  • mutations：都是函数类型，只需要指定参宿和类型即可。注意只能有一个参数，如果多个需要封装成对象。changeOpen：function（state：State，params：{a：string，b：string，c：number}）：void {}
