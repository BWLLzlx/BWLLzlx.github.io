# 文档流

# 标准流

标签按照规定好的默认方式排列，最常用于纵向布局

# 浮动流

常用于横向布局，创建浮动框将其移到左右边缘或者另一个浮动框的边缘

特性：先浮后动（浮起来底下的位置也空了，所以后边的没浮动的也会受影响往上走，没浮的理解为是一堵前低后无限高的墙，墙后面的无论如何都没办法到这墙前面）、浮动元素顶部对齐，行内块（基本还是行内元素，不过能调大小）

通常是纵向用文档流先布局完，然后每个div里用浮动布局。而且要浮动就全浮动

浮动会压住后面的标准流元素，但是不会压住文字

清除浮动：如果根据子元素的内容的内容设定父元素高度，当子元素浮动后，子元素没东西撑开，高度就会变成0，父元素没有浮动，会影响下面的元素。所以需要清除浮动（清除浮动对标准流的影响）

  • 方法1：在最后一个浮动元素后添加块元素当一堵墙，添加清除浮动，可惜会添加过多冗余结构
  • 方法2：给父元素设置overflow:hidden，可惜无法显示溢出的部分
  • 方法3：after伪元素法，和方法1一样，只是结构中没有，但css有
  • 方法4：双伪元素，：：after和：：before都用上

定位用于让盒子在某个盒子内移动位置或者在屏幕中固定某个位置，并且可以压住其他盒子，设置叠放层次靠z-index设置z轴，值相同则下面的上去

绝对、固定定位什么都给下面的压完了

# 相对定位

相对于自己原来的位置移动，自己走了，但是位置还在

# 绝对定位

相对于自己的祖先元素，如果没有祖先或者祖先没有定位，就以浏览器定位；如果祖先有定位，取最近一级，自己走了，位置也走了（理解为浮动，盒子 也变成了行内块的特性）

# 固定定位

相对于浏览器可视窗口，滚动条不影响。跟父元素没关系。不占用位置（理解为特殊的绝对定位）。设置固定定位常常通过百分比到一个位置例如50%， 然后再加版心的一半。或者先定位到50%，然后减掉自己的一半（居中）

# 粘性定位

基本上是相对定位，但是当移动到距离设置的某一条边一定距离的时候，就会黏住不走，变成固定定位

# BFC和IFC

FC是format context，格式化上下文，是页面中的一块渲染区域，并且规定了元素如何定位以及和其他元素的相互关系。包括块格式化上下文BFC和内联格式化上下文IFC

# BFC

BFC属于普通流，是一块渲染区域，有独立的渲染规则：①盒子垂直放置，②同一个BFC内上下margin会重叠，③盒子的左边与子盒子的左边接触，即使浮动，④BFC不会与float重叠，⑤有隔离作用，容器内的子元素不会影响到外面的元素，⑥计算BFC的高度时浮动的也要算

# 如何产生BFC

  • 根元素
  • float不是none
  • 绝对定位或固定定位
  • display为非块的块级元素例如inline-block
  • overflow不是visible

# BFC的用途

  • 解决嵌套块元素外边距塌陷（放在两个不同的BFC里，margin就不会重叠了，②⑤）
  • 清除浮动（当父元素为BFC时，父元素能够被浮动元素撑开，⑥）
  • 阻止被浮动元素覆盖（设置为BFC后，BFC就不会和float重叠了，④⑤）

# flex弹性布局

任何容器（块、行内元素）都变成行内块的形式。设置了flex之后，float就没用了，也不需要clear了，也可以垂直居中，不需要vertical-align

flex布局中，默认只有一条轴，不换行，如果内容装不下就会缩小子元素

  • 父亲元素设置flex属性：flex-direction主轴方向（row、row-reverse、column）、justify-content在主轴的排列方式（center挤在中间、flex-start挤在起点、flex-end挤在终点、space-around平分剩余空间、space-between贴到两边后平分剩余空间）、flex-wrap是否换行、align-content侧轴子元素多行排列（需要先允许换行，和主轴差不多，单行用无效）、align-items侧轴子元素排列只有单行（center等，和主轴差不多，还有一个stretch但是不能给高度，没有平分剩余空间，因为单行的情况下侧轴只有1个东西，不存在平分）

  • 子元素设置flex属性：空间不足时是否缩短、空间足够时是否拉长、占据剩余空间的权重，只有自己就是全占。align-self：控制元素自己的位置。order属性，定义子元素的顺序，默认是0，-1比0小就在前面

# grid网格布局

根据行轨道、列轨道划分出若干区域

给外层容器设置display：grid，直接子元素就会变成grid布局。

划分区域：grid-template-columns属性自由划分区域，设置宽度如100px 100px 200px ，也可以用权重单位fr。column-gap属性设置间距。

分配区域：grid-template-areas：假如一行分了3个区域，grid-template-areas：“header header header”表示三个区域都是header的部分，如果由多行每一行分开一个引号，中间不需要别的，如“header header header”“aside main main”。每个组件用grid-area指定网格区域的名字。也可以直接每个部分划分：用grid-column、row-start、end指定每一个区域的范围，从1开始。

区域内对齐：align-items纵向，justify-items横向，和flex很像。要全部缩在一起就align-content，justify-content。指定单个：align-self、justify-self

# 媒体查询

媒体查询：@media针对不同的屏幕尺寸设置不同的样式，媒体查询从小到大写（因为下面会把上面覆盖掉）

媒体查询语法：@media 媒体类型（all|screen|print） and|not|only 媒体特性（max-width、min-width），例如：如果是在屏幕上并且最大宽度=800像素，就去设置里面的内容。通过rem，将屏幕宽度分类成不同的情况然后各做设置。也可以根据屏幕宽度引入不同的css文件，link有个属性是media，就像写media语法一样

less：css弊端不利于维护、冗余度高、无法计算。less是一门css拓展语言，也是css预处理器。less在css的基础上引入了变量、运算、函数，简化编写方便维护。less在vscode里面用插件可以被编译为css文件，页面引用的还是css文件

  • 变量：less文件的变量需要使用@
  • 混合：某个选择器u内写了x属性：X，y属性：Y。可以通过u（）在其他选择器内使用u选择器内的属性值设置
  • 嵌套：子元素的样式写在父元素内部。伪元素、交集、伪类需要在内层加&，否则会被编译成子元素的样式。也就是说**&指向父元素，有&就是写给父元素的**
  • @嵌套：@规则与选择器同样嵌套。内部顺序不变
  • 运算：less运算符的两侧必须要空格隔开，两个数只有一个单位参与运算以有的为准，多个有单位以第一个为准，换算后无效就忽略单位。
  • calc（）：不去计算具体运算结果，但是会把变量和公式算出来
  • 转义：可以把任意字符串作为变量的值或属性。例如@abc = ~“min-width：300px”。那么@media @abc{}也是能用的，也可以@abc = (min-width：300px)
  • 命名空间：为了组织封装、避免重名，对选择器进行分组，例如#a {}命名空间里面加入.b{}、.c{}，想混合c的时候就#a.c（）
  • 映射：可以通过下标运算[]来获取混合的值，毕竟是键值对
  • 作用域：作用域链，先在当前作用域找，找不到就从父级的作用域继承。和js的预加载类似，会有变量提升
  • 导入：引用别的less文件：@import “a”；

操作思路：实现等比例缩放，把屏幕划分成若干等分（10、15、20），这个数量也是用变量，一等分的宽度就是1rem，不同的屏幕宽度下rem就不同，但是页面比例相同

操作步骤：先设置rem的大小和划分的份数no。媒体查询设置不同屏幕宽度下的rem（html的font-size：对应宽度 px / no）。设置body的最小宽度和最大宽度（如果用flexible.js自动计算出rem十分之一屏幕宽度，这时候宽度再设计成等分总数的rem就会占满屏幕，还是需要max-width，可以用媒体查询设置“min-width：最大宽度”的样式rem固定并且加上!important就行，覆盖掉flexible.js计算的结果），如果要设置宽度就设置成例如15rem，然后可以用margin。内部盒子的宽高都要转化为rem作单位

# 3D转换

操作

  • perspective：设置视距。开启透视，写在被观察标签的父标签，模拟人的眼睛到屏幕的距离，想要有3d效果必须有透视，谁会动并且父标签不会动就给那个父标签

  • transform-style：设置子元素是否以3D效果呈现。默认情况下是flat，3D转换父元素的时候，子元素会变成平面。设置为preserve-3d就能

  • transform：translate3d（x，y，z）z轴的位移量通常只能写px

  • transfom：rotateX、Y、Z（deg）绕着某一条轴旋转

实例

  • 实现正反面翻转：相当于翻转一个骰子，1看到4。外层div（骰子）内有两个div，一个正面（1），一个反面（4），反面（4）需要自己绕着y轴旋转180度（内容翻转，当骰子翻转时看到的就负负得正）。因为需要看起来有3d效果，所以需要给外层div（骰子）设置perspective。而且因为正反面翻转是旋转外部的div，所以需要在操作父元素的时候保留子元素的3d效果transform-style：preserve-3d，否则4就白翻转了
  • 实现90度翻转：关键点在于知道翻转的轴在哪。其中一个div翻转后是X形，要变成L形就需要平移。平移后应该要让旋转轴经过两个div的中垂线交点处，而不是在某个平面上
