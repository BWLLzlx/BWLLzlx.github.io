# 基础部分

# 为什么需要打包工具？

开发时，我们会使用框架（React、Vue），ES6 模块化语法，Less/Sass 等 css 预处理器等语法进行开发

这样的代码要想在浏览器运行必须经过编译成浏览器能识别的 JS、Css 等语法，才能运行。需要打包工具帮我们做完这些事

# 为什么要配置打包工具

webpack只能做js处理，配置webpack主要就是像要想处理其他资源。并且添加一些设定，如打开缓存、压缩代码、代码分割、做兼容性处理，最终的目标就是提升代码性能

# 常见的打包工具

  • Parcel
  • Webpack
  • Vite

# 使用webpack

  • 下载依赖

    初始化package.json：npm init -y

    下载依赖：npm i webpack webpack-cli -D

  • 启用 webpack

    开发模式：npx webpack ./src/main.js --mode=development

    生产模式：npx webpack ./src/main.js --mode=production

  • 查看输出

    默认 webpack 会将文件打包输出到 dist 目录下，我们查看 dist 目录下文件情况就好了

# webpack基本配置

webpack 是基于 Node.js 运行的，所以采用 Common.js 模块化规范

  • entry（入口）

    指示 Webpack 从哪个文件开始打包。通常就是一个main.js

  • output（输出）

    指示 Webpack 打包完的文件输出到哪里去，如何命名等。通常需要引入path模块

    output: {
    path: path.resolve(__dirname, "dist"),
    filename: "main.js",
},

  • loader（加载器）

    webpack 本身只能处理 js、json 等资源，处理其他资源需要借助 loader，Webpack 才能解析

  • plugins（插件）

    扩展 Webpack 的功能，例如缓存、处理HTML模板、压缩代码

  • mode（模式）

    • 开发模式：development

  • 生产模式：production

# 修改输出资源的名称和路径

//当前用的是生产环境下的配置

const path = require("path");

module.exports = {
  entry: "./src/main.js",
  output: {
    path: path.resolve(__dirname, "../dist"),
    filename: "static/js/[name].[contenthash:10].js"// 将 js 文件输出到 dist/js 目录中，用contenthash添加缓存
    chunkFilename:"static/js.[name].[contenthash:10].chunk.js",// 打包的chunk也要命名，命名成name.chunk.js
    assetModuleFilename: "static/media/[hash:10][ext][query]", //命名图片
  },
  module: {
    rules: [
    ],
  },
  plugins: [],
  mode: "production",
};

# 自动清空上次打包资源

当我们进行打包后，会生成一个dist目录，但是当我们再一次进行打包时，上一次生成的dist目录并不会清空，所以我们需要手动删除上一次的dist目录

output: {
    clean: true, // 自动将上次打包目录资源清空
},

起初我们想要查看打包好的文件，都是通过点击打开dist目录下的index.html，通过插件open with live server 在浏览器中查看。插件webpack-dev-server可以实现自动化，我们可以看到每次打包好的页面

下载：npm i webpack-dev-server -D

// 开发服务器

    open: true, // 是否自动打开浏览器

安装插件：npm install html-webpack-plugin -D

调整 webpack.config.js 文件：在插件中加入new HtmlWebpackPlugin() 去实例化 html-webpack-plugin 插件，发现这个 dist/index.html 似乎与先前的 index.html 并没有关系，HtmlWebpackPlugin 会默认生成它自己的 index.html 文件，并且所有的 bundle（bundle.js） 会自动添加到 html 中，所以要配置以原本的html作为模板

# 处理样式资源

webpack 本身是不能识别样式资源的，所以我们需要借助 loader 来帮助 webpack 解析样式资源

# CSS

安装插件：

  • css-loader：负责将 Css 文件编译成 Webpack 能识别的模块
  • style-loader：会动态创建一个 Style 标签，里面放置 Webpack 中 Css 模块内容

配置：

module: {
    rules: [
      {
        // 用来匹配 .css 结尾的文件
        test: /\.css$/,
        // use 数组里面 Loader 执行顺序是从右到左
        use: ["style-loader", "css-loader"],
      },
    ],
  },

需要在入口文件引入css资源，Webpack才会对其打包

# Less

下载：npm i less-loader -D

功能介绍：负责将 Less 文件编译成 Css 文件

module: {
    rules: [
      {
        test: /\.less$/,
        use: ["style-loader", "css-loader", "less-loader"],
      },
    ],
},
# 提取CSS成单独文件（生产模式）

Css 文件如果用上面的方式，是先打包到 js 文件中，当 js 文件加载时，再创建一个 style 标签来生成样式，会出现闪屏现象，用户体验不好，原因就是js引擎和渲染进程的互斥。所以需要提取成单独的 Css 文件，通过 link 标签加载性能才好

下载：npm i mini-css-extract-plugin -D

配置：

const MiniCssExtractPlugin = require("mini-css-extract-plugin");

plugins: [
    // 提取css成单独文件
    new MiniCssExtractPlugin({
      // 定义输出文件名和目录
      filename: "static/css/main.css",
    }),
],

# 处理js资源

js 资源 Webpack 不是已经处理了吗，为什么我们还要处理呢？ Webpack 对 js 处理是有限的，只能编译 js 中 ES 模块化语法，不能编译其他语法，导致 js 不能在 IE 等浏览器运行，所以我们希望做一些兼容性处理。其次，团队对代码格式是有严格要求的，我们不能由肉眼去检测代码格式

# Eslint

eslint就是用来限制代码格式的

下载包：npm i eslint-webpack-plugin eslint -D

Eslint 配置文件：位于项目根目录，配置文件可以有多种写法，.eslintrc、.eslintrc.js、.eslintrc.json。ESLint 会查找和自动读取，只需要存在一个即可

module.exports = {
  // 继承 Eslint 规则。开发中一点点写 rules 规则太费劲了，所以有更好的办法，继承现有的规则
  extends: ["eslint:recommended"], // 如果要vue的话就多一个plugin:vue/vue3-essential
  env: {
    node: true, // 启用node中全局变量
  },
  parserOptions: {
    ecmaVersion: 6,// ES 语法版本
    sourceType: "module",// ES 模块化
  },
  rules: {
    //"off" 或 0 - 关闭规则
	//"warn" 或 1 - 开启规则，使用警告级别的错误：warn (不会导致程序退出)
	//"error" 或 2 - 开启规则，使用错误级别的错误：error (当被触发的时候，程序会退出)
    "no-var": 2, // 不能使用 var 定义变量
  },
};

webpack配置：

 plugins: [
    new ESLintWebpackPlugin({
      	// 指定检查文件的根目录
     	context: path.resolve(__dirname, "../src"),
        exclude: "node_modules", //默认值就是这个,也就是忽略node_modules下的内容
        //开启缓存
        cache: true, //打开缓存
        cacheLocation: path.resolve(__dirname, "../node_modules/.cache/.eslintcache"),//设置缓存 路径
    }),
  ],

# 处理图片资源

Webpack5 已经将 file-loader 和 url-loader两个 Loader 功能内置到 Webpack 里了，只需要简单配置即可处理图片资源

配置：

module: {
    rules: [
      {
        test: /\.(png|jpe?g|gif|webp|svg)$/,
        type: "asset",
        //可以不加
        parser: {
            dataUrlCondition: {
                maxSize: 10 * 1024 // 小于10kb的图片会被base64处理，优点减少请求数量，缺点体积变大
            }
        },
      },
    ],
  },

type: "asset/resource"和type: "asset"的区别：

  1. type: "asset/resource" 相当于file-loader, 将文件转化成 Webpack 能识别的资源，其他不做处理
  2. type: "asset" 相当于url-loader, 将文件转化成 Webpack 能识别的资源，同时小于某个大小的资源会处理成 data URI 形式

# 进阶

# 代码分离

可以把代码分离到不同的 bundle 中，然后可以按需加载或并行加载这些文件。常用的代码分离方法有三种

# 多入口起点

方法：使用 entry 配置手动地分离代码

下载：在 src 目录下创建 another-module.js 文件，/这个模块依赖了 lodash ，需要安装一下： npm install lodash --save-dev

配置：

//another-module.js

import _ from 'lodash'
console.log(_.join(['Another', 'module', 'loaded!'], ' '))

配置webpack：

module.exports = {
    entry: {
   		index: './src/index.js',
    	another: './src/another-module.js',
	},
    output: {
    	filename: '[name].bundle.js'//name表示以文件名自己命名
    },
}

打包后的文件：两个入口的 bundle 文件都被链接到了 app.html 中

缺点：

  • 不够灵活，不能动态地将核心应用程序逻辑中的代码拆分出来
  • 如果入口 chunk 之间包含一些重复的模块，那些重复模块都会被引入到各个 bundle 中

SPA单入口也不需要这个

# 防止重复

通常是用SplitChunksPlugin 可以将公共的依赖模块提取到已有的入口 chunk 中，或者提取到一个新生成的 chunk

module.exports = {
    optimization: {
        //打包后的一份代码都可以称为chunk
       splitChunks: {
           chunks: 'all',
           //其他配置都用默认值
           //可能需要修改的配置有
           //cacheGroups，指定哪些模块打包到同一个组
           //minSize，最小的需要打包的文件。可以设置成0，让再小的文件也会被打包
       },
    },
}

# 动态导入

使用符合 ECMAScript 提案的 import() 语法 来实现动态导入

# 懒加载

懒加载是一种很好的优化网页或应用的方式。把 你的代码在一些逻辑断点处分离开，然后在一些代码块中完成某些操作后，立即引用 或即将引用另外一些新的代码块。这样加快了应用的初始加载速度，减轻了它的总体 体积，因为某些代码块可能永远不会被加载

案例：

一个功能模块

//math.js
export const add = () => {
	return x + y
}
export const minus = () => {
	return x - y
}

使用：

const button = document.createElement('button')
button.textContent = '点击执行加法运算'
button.addEventListener('click', () => {
    import(/* webpackChunkName: 'math' */ './math.js').then(({ add}) => {//解构赋值拿出add
   		console.log(add(4, 5))
    })
})
document.body.appendChild(button)

加上 webpackChunkName: 'math' , 告诉webpack打包生成的文件名为 math

第一次加载完页面， math.bundle.js 不会加载，当点击按钮后，才加载 math.bundle.js 文件

eslint不能识别动态导入，需要在plugin插件中配置、添加一个“import”

# 预获取/预加载

  • prefetch(预获取)：将来某些导航下可能需要的资源。加上webpackPrefetch: true，告诉 webpack 执行预获取，这会生成<link rel="prefetch" href="math.js">并追加到页面头部，指示着浏览器在闲置时间预取 math.js 文件

  • preload(预加载)：当前导航下可能需要资源

    • 在父 chunk 加载时**，以并行方式开始加载**。prefetch chunk 会在父 chunk 加载结束后开始加载
    • preload chunk 具有中等优先级，并立即下载

# 缓存

浏览器通过缓存减少资源请求时间。这样也会带来一些问题：在部署新版本时如果不更改资源的文件名，浏览器可能会认为它没有被更新，就 会使用它的缓存版本

所以需要通过webpack进行配置确保：

  • 编译生成的文件能够被客户端缓存
  • 文件内容变化后，浏览器能够请求到新的文件

# 修改输出文件的文件名

[contenthash] 将根据资源 内容创建出唯一 hash，当资源内容发生变化时， [contenthash] 也会发生变化。内容 -》 hash -》bundle名称

module.exports = {
    output: {
    	filename: '[name].[contenthash].js',
    },
};

# 拆分开发环境和生产环境

之前只能通过在webpack的配置文件中手动选择环境，且很多 配置在生产环境和开发环境中存在不一致的情况，比如开发环境没有必要设置缓存， 生产环境还需要设置公共路径等等

# 环境变量

想要消除 webpack.config.js 在 开发环境 和 生产环境 之间的差异，需要环境变量。webpack 命令行 环境配置的 --env 参数，允许传入任意数量的环境变量，而在 webpack.config.js 中可以访问到这些环境变量，也就是说可以在webpack配置文件中获取到命令行输入的那个参数

module.exports = (env) => {
    return {
    //...
    // 根据命令行参数 env 来设置不同环境的 mode
    mode: env.production ? 'production' : 'development',
    //...
    }
}

# 优化

# source-map

通过webpack, 将我们的源码打包成了 bundle.js，客户端(浏览器)读取的是打包后的 bundle.js ,那么当浏览器执行代码报错的时候，报错的信息自然也是bundle的内容，webpack已经内置了sourcemap的功能，将报错信息及其所在行列)映射到源码上，用于debug

配置：

module.exports = {
    // 开启 source map
    // 生产环境中推荐使用 'source-map'
    // 开发环境用 cheap-module-source-map
    devtool: 'source-map',
}

生产环境我们一般不会开启sourcemap功能

  • 会暴露源码
  • sourcemap文件的体积相对比较巨大,这跟我们生产环境的追 求不同(生产环境追求更小更轻量的bundle)

# Include/Exclude

第三方的库或插件，所有文件都下载到 node_modules 中了。而这些文件是不需要编译可以直接使用的。在对 js 文件处理时，要排除 node_modules 下面的文件

  • js资源的处理用include，获取path.resolve(__dirname, "../src")
  • 像eslint就可以用exclude: "node_modules",
# cache

每次打包时 js 文件都要经过 Eslint 检查 和 Babel 编译，速度比较慢。可以缓存之前的 Eslint 检查 和 Babel 编译结果

module.exports = {
  module: {
    rules: [
      {
        oneOf: [
          {
            test: /\.js$/,
            // exclude: /node_modules/, // 排除node_modules代码不编译
            include: path.resolve(__dirname, "../src"), // 也可以用包含
            loader: "babel-loader",
            options: {
              cacheDirectory: true, // 开启babel编译缓存
              cacheCompression: false, // 缓存文件不要压缩
            },
          },
        ],
      },
    ],
  }, 
};

//ESLint的在插件中配置
new ESLintWebpackPlugin({
    // 指定检查文件的根目录
    context: path.resolve(__dirname, "../src"),
    exclude: "node_modules", //默认值就是这个
    cache: true, //打开缓存
    cacheLocation: path.resolve(__dirname, "../node_modules/.cache/.eslintcache"),//设置缓存 路径
}),

# 减少代码体积

开发时我们定义了一些工具函数库，或者引用第三方工具函数库或组件库，如果没有特殊处理的话我们打包时会引入整个库，但是实际上可能我们可能只用上极小部分的功能。Webpack 已经默认开启了Tree Shaking功能，移除 JavaScript 中的没有使用上的代码

sideEffects：有些模块只要被引入，就会对应用程序产生重要的影响，这就是副作用，如何告诉 Webpack 你的代码无副作用，可以通过 package.json 有一个特殊的属性 sideEffects，值可以是true、false、数组

还可以压缩图片，使用image minisizer

# Network Cache

因为浏览器会使用缓存，如果前后版本的文件名没有变化导致浏览器直接读取缓存，不加载新资源，项目没法更新。解决办法：contenthash，根据文件内容生成 hash 值，只有文件内容变化了，hash 值才会变化。上面也提到过

# Core-js

babel 对 js 代码进行了兼容性处理，能将 ES6 的一些语法进行编译转换，比如箭头函数、点点点运算符等。但是如果是 async 函数、promise 对象、数组的一些方法（includes）等，它没办法处理。

core-js 是专门用来做 ES6 以及以上 API 的 补丁

下载：npm i core-js

引用：import "core-js/es/promise"; //手动按需引入， import "core-js";是全部引入，体积较大

配置：

module.exports = {
  // 智能预设：能够编译ES6语法
  presets: [
    [
      "@babel/preset-env",
      // 按需加载core-js的polyfill
      { useBuiltIns: "usage", corejs: { version: "3", proposals: true } },
    ],
  ],
};

# webpack配置

//webpack.config.js


const path = require("path");
const HtmlWebpackPlugin = require("html-webpack-plugin");
const ESLintWebpackPlugin = require("eslint-webpack-plugin");
const MiniCssExtractPlugin = require("mini-css-extract-plugin");
const CssMinimizerPlugin = require("css-minimizer-webpack-plugin");
const TerserPlugin = require("terser-webpack-plugin");
const { VueLoaderPlugin } = require("vue-loader")
const { DefinePlugin } = require("webpack")
const CopyPlugin = require("copy-webpack-plugin");
const AutoImport = require('unplugin-auto-import/webpack')
const Components = require('unplugin-vue-components/webpack')
const { ElementPlusResolver } = require('unplugin-vue-components/resolvers')

const isProduction = process.env.NODE_ENV === "production"

//因为less、sass、css都有这一段，所以提取出来。并且因为各自都有自己需要的loader，所以还要传入
const getStyleLoaders = (loader) => {
    return [
        isProduction ? MiniCssExtractPlugin.loader : "vue-style-loader",
        "css-loader",
        {
            loader: "postcss-loader", // 用于解决样式的兼容性问题，还需要配置样式兼容性做到什么程度，需要在package.json配置browserslist
            options: { // 需要指定选项
                postcssOptions: {
                    plugins: ["postcss-preset-env"], // 指定插件
                }
            }
        },
        loader
    ].filter(Boolean)//用于过滤空值，因为css没有loader
}

module.exports = {
    // 入口
    // 相对路径和绝对路径都行
    entry: "./src/main.js",
    // 输出
    output: {
        path: isProduction ? path.resolve(__dirname, "../dist") : undefined,
        filename: isProduction ? "static/js/[name].[contenthash:10].js" : "static/js/[name].js", // 将 js 文件输出到 dist/js 目录中，用contenthash添加缓存
        chunkFilename: isProduction ? "static/js.[name].[contenthash:10].chunk.js" : "static/js.[name].chunk.js",// 打包的chunk也要命名，命名成name.chunk.js
        assetModuleFilename: "static/media/[hash:10][ext][query]", //命名图片
        clean: true, // 自动将上次打包目录资源清空
    },
    // 加载器
    module: {
        rules: [
            //js的babel，同时还要配置babel.config.js
            {
                test: /\.js$/,
                include: path.resolve(__dirname, "../src"),
                // exclude: /node_modules/,//排除这里面的不需要babel进行处理
                loader: 'babel-loader',
                options: {
                    cacheDirectory: true, // 开启babel编译缓存
                    cacheCompression: false, // 缓存文件不要压缩，打包速度更快
                },
            },
            //css
            {
                test: /\.css$/,
                // 最终的操作：把样式，通过动态创建style标签插入到页面而生效
                use: getStyleLoaders(),
            },
            //less
            {
                test: /\.less$/,
                use: getStyleLoaders("less-loader"),
            },
            //sass
            {
                test: /\.s[ac]ss$/,//两种写法，sass和scss
                use: getStyleLoaders("sass-loader"),
            },
            //图片
            {
                test: /\.(png|jpe?g|gif|webp|svg)$/,
                type: "asset",
                parser: {
                    dataUrlCondition: {
                        maxSize: 10 * 1024 // 小于10kb的图片会被base64处理，优点减少请求数量，缺点体积变大
                    }
                },
            },
            //字体等多媒体类型
            {
                test: /\.(ttf|woff2?|map3|map4|avi)$/,
                type: "asset/resource", //asset和这个的区别是：asset可以转base64，这个不行
            },
            // vue，还需要引入插件、和解决环境变量
            {
                test: /\.vue$/,
                loader: "vue-loader",
                options: {
                    cacheDirectory: path.resolve(__dirname, "../node_modules/.cache/vue-loader"), // 想快就开缓存
                }
            }
        ],
    },
    // 插件
    plugins: [
        new ESLintWebpackPlugin({
            // 指定检查文件的根目录
            context: path.resolve(__dirname, "../src"),
            exclude: "node_modules", //默认值就是这个
            cache: true, //打开缓存
            cacheLocation: path.resolve(__dirname, "../node_modules/.cache/.eslintcache"),//设置缓存 路径
        }),
        //html插件
        new HtmlWebpackPlugin({
            template: path.resolve(__dirname, "../public/index.html"),//以这个为模板生成index.html
        }),
        new VueLoaderPlugin(),
        //cross-env定义的环境变量是给webpack打包工具用的
        //definePlugin定义的环境变量是代码使用的，解决Vue3的警告
        new DefinePlugin({
            __VUE_OPTIONS_API__: true,
            __VUE_PROD_DEVTOOLS__: false,
        }),
        //抽取独立css
        isProduction && new MiniCssExtractPlugin({
            // 定义输出文件名和目录
            filename: "static/css/[name].[contenthash:10].css",
            chunkFilename: "static/css/[name].[contenthash:10].chunk.css",
        }),
        //把public文件中的资源复制到dist中
        isProduction && new CopyPlugin({
            patterns: [
                {
                    from: path.resolve(__dirname, "../public"),
                    to: path.resolve(__dirname, "../dist"),
                    globOptions: {
                        //忽略这个文件
                        ignore: ["**/index.html"],
                    }
                },
            ],
        }),
        //Element Plus 按需引入
        AutoImport({
            resolvers: [ElementPlusResolver()],
        }),
        Components({
            resolvers: [ElementPlusResolver()],
        }),

    ].filter(Boolean),
    // 模式
    mode: isProduction ? "production" : "development", // 生产模式，js会自动压缩
    devtool: isProduction ? "source-map" : "cheap-module-source-map", // 开发模式的调试，开启source-map调试
    optimization: { // 打包成多个chunk而不是一个js文件，也就是代码分割
        splitChunks: {
            chunks: "all",
            //将不同的文件打包到不同的地方
            cacheGroups: {
                vue: {
                    test: /[\\/]node_modules[\\/]vue(.*)?[\\/]/,
                    name: "vue-chunk",
                    priority: 40,
                },
                elementPlus: {
                    test: /[\\/]node_modules[\\/]element-plus(.*)?[\\/]/,
                    name: "elementPlus-chunk",
                    priority: 30,
                },
                libs: {
                    test: /[\\/]node_modules[\\/]/,
                    name: "libs-chunk",
                    priority: 20,
                }

            }
        },
        //代码分割可能导致缓存失效，配置这个
        runtimeChunk: {
            name: entrypoint => `runtime~${entrypoint.name}.js`,
        },
        minimize: isProduction,
        minimizer: [
            //css压缩插件
            new CssMinimizerPlugin(),
            //做了css压缩的话，还要做js压缩，否则有点问题
            new TerserPlugin(),
        ]
    },
    // webpack解析模块加载选项
    resolve: {
        // 自动补全文件扩展名
        extensions: [".vue", ".js", ".json"],
        // 补充优化：路径别名
        alias: {
            "@": path.resolve(__dirname, "../src")
        }
    },
    //自动化构建dist和自动启动服务
    devServer: {
        host: "localhost", // 启动服务器域名
        port: "3000", // 启动服务器端口号
        open: true, // 是否自动打开浏览器
        hot: true, //热模块
        historyApiFallback: true, //解决前端路由刷新404
    },
    performance: false, // 关闭性能分析，提升打包速度，优化
};
